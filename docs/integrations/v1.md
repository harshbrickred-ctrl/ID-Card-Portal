# Integration v1 — ID Card Portal

This document defines how any SaaS app (Vetan HR, School ERP, etc.) connects to the standalone ID Card Portal.

## Overview

1. **SSO handoff** — User clicks "ID Cards" in your app; browser opens the portal with a short-lived JWT.
2. **Employee sync** — Portal calls your API with a tenant-scoped integration API key to pull employee data for printing.

## SSO handoff (your app → portal)

Redirect the user's browser to:

```
GET {PORTAL_URL}/auth/connect?source={source}&token={jwt}
```

### JWT claims (HS256, 5 minute TTL)

Sign with the shared secret configured for your `source` key in portal `JWT_SECRETS`.

```json
{
  "source": "vetan",
  "externalOrgId": "tenant-uuid",
  "userId": "user-uuid",
  "email": "admin@company.com",
  "name": "Admin User",
  "permissions": ["id-cards:read", "id-cards:write"],
  "orgName": "Acme Pvt Ltd",
  "apiBaseUrl": "https://app.vetan.app"
}
```

## Employee sync (portal → your app)

Your app exposes:

```
GET /v1/integrations/id-cards/employees
Authorization: Bearer {integration_api_key}
```

### Response

```json
{
  "schemaVersion": "1",
  "organization": {
    "externalId": "tenant-uuid",
    "name": "Acme Pvt Ltd",
    "code": "ACME",
    "logoUrl": null
  },
  "employees": [
    {
      "externalId": "employee-uuid",
      "employeeCode": "EMP001",
      "firstName": "Raj",
      "lastName": "Sharma",
      "email": "raj@acme.com",
      "department": "Engineering",
      "designation": "Software Engineer",
      "status": "ACTIVE",
      "dateOfJoining": "2024-01-15",
      "photoUrl": "https://your-api/.../signed-photo-url"
    }
  ]
}
```

### Photo URLs

- Return HTTPS URLs with at least **15 minutes** TTL.
- `null` if no photo; portal will flag missing photos before export.

## API key management

- Issue per-tenant integration keys in your SaaS settings UI.
- Store only a hash in your database; show the raw key once on creation.
- Recommended prefix: `vic_` for Vetan, `sic_` for School ERP.

## CR-80 export (portal)

Portal exports a ZIP per batch:

- `{employeeCode}_front.png` — 1011×638 px (300 DPI CR-80)
- `{employeeCode}_back.png` — QR + company info on back
- `manifest.csv` — index of all files

Import the ZIP into Evolis, Zebra, or HID badge software.

## Adding a new SaaS source

1. Add your `source` string to portal `JWT_SECRETS`.
2. Implement the sync endpoint above in your app.
3. Add SSO link + API key settings in your app UI.
